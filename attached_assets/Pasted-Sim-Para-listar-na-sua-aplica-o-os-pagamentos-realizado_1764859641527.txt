Sim. Para listar na sua aplicação os pagamentos realizados via Checkout, você precisa listar pedidos ou cobranças na API e usar algum identificador que relacione esses pedidos ao Checkout (por exemplo, o próprio `checkout_link_id`, `code` ou `metadata`).[1][2][3]

## Endpoints relevantes

- O Checkout em si é apenas a “porta de entrada”; os pagamentos realizados ali viram `orders` e `charges` na API Core (`/core/v5`).[2][3]
- Para trazer esses pagamentos para dentro da sua aplicação você usa principalmente:
  - `GET /core/v5/orders` → listar pedidos com filtros por status, data, código etc.[4][1]
  - `GET /core/v5/charges` → listar cobranças com filtros por status (`pending`, `paid`, `canceled`, `failed`, `overpaid`, etc.).[5][6]

## Como diferenciar “pagos no Checkout”

- A página de Checkout orienta que você crie o Link de Pagamento via API de Checkout (`/checkout`) e receba uma URL para enviar ao cliente; esse link costuma estar associado a um pedido ou pode carregar um `code`/`metadata` que você define.[3][2]
- Na hora de criar o pedido ou o link, defina um identificador (ex.: `code: "checkout_site"`, ou um campo em `metadata`) e depois use esse campo como filtro nos endpoints de `orders`/`charges` para listar apenas pagamentos que vieram desse Checkout.[1][5]

## Fluxo típico na sua aplicação

- Quando criar o Checkout/Link, armazene no seu banco: `order_id` (se já existir), algum `code` próprio e a URL do checkout; assim você consegue relacionar futuramente a cobrança ao seu pedido interno.[2][3]
- Para listar na tela “Pagamentos realizados no Checkout”, sua API faz:
  - Uma chamada periódica ou sob demanda para `GET /core/v5/orders` ou `GET /core/v5/charges` filtrando por:
    - status = `paid` (ou os que fizerem sentido para você)  
    - e, opcionalmente, `code`/`metadata` que você usou para marcar o pedido como vindo do Checkout.[5][1]
